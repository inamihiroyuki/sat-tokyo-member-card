# 東京都スキー連盟 会員マイページ / デジタル会員証モック 議論メモ

作成日: 2026-06-20

## 目的

東京都スキー連盟の新しい会員向けWebシステムについて、受注獲得のためにUXイメージを説明できるHTMLモックを作成した。

モックの主目的は、以下の流れを商談相手に直感的に見せること。

- 東京都スキー連盟会員ログイン
- 東京都スキー連盟会員番号と携帯電話番号の入力
- 携帯電話に届いた6桁コードによるログイン
- 東京都スキー連盟会員マイページ
- デジタル会員証の表示
- Appleウォレット / Googleウォレットへの登録

## システム方針

これまでの議論では、既存の古いPHP業務システムは置き換えず、既存業務DBを正本として維持する方針とした。

新システムは以下の前提で整理した。

- 既存業務システム本体はなるべく触らない
- 既存業務DBはREAD ONLY専用ユーザーで直接参照する
- 新システムから既存業務DBへ書き込みはしない
- 新システム用DBには、新システム固有情報だけを保存する
- 会員向けには、マイページ、携帯電話ログインコード、デジタル会員証、Wallet登録を提供する

新システムDBに保存する想定の情報:

- ログインコード送信履歴
- ログイン履歴
- コード入力失敗回数
- 顔写真
- Wallet発行情報
- QR照会用トークン
- 利用規約同意履歴
- アクセスログ
- 監査ログ

- 携帯電話番号

ボタン:

方針:

- 高齢会員にも分かりやすいよう、画面上では原則として `SMS` という言葉を使わない
- `携帯電話に届いた6桁の数字` や `ログインコード` という表現にする

- `会員番号とSMSだけで、会員証まで進める。`

### Wallet登録

説明文:

`東京都スキー連盟会員のデジタル会員証をスマートフォンに登録すると、紛失の心配も無く、紙のカードと同じようにご利用頂けます。`

ボタン:

方針:

- 両方とも同じ黒い横長ボタンデザインに統一
- 文言は `登録する` に統一
- Apple側は公式バッジ素材の利用許諾同意が必要だったため、公式素材ではなくモック用の黒ボタンとして実装
- Google側も文言統一のため、公式SVG画像ではなく同デザインの黒ボタンに変更

## モック注釈の扱い

設計上のポイントは、実際の会員向けUIとは別コンテクストであることを明確にするため、オレンジ枠の `モック注釈` として表示する。

仕様:

- オレンジ枠
- `モック注釈` ラベル付き
- ゆっくり上下に浮かぶアニメーション
- `prefers-reduced-motion: reduce` ではアニメーション停止

用途:

- パスワード不要の説明
- 既存DBをREAD ONLYで参照する説明
- 新DBに保存する情報の説明
- Wallet発行APIやQR照会トークンの説明

## 検証済み事項

- PC幅でログイン画面を確認
- スマホ幅でログイン画面を確認
- ログインボタン押下でコード入力モーダルが開く
- 6桁コード入力画面からマイページへ遷移
- マイページで都連花子と顔写真が表示される
- Wallet登録ボタンがスマホ幅で1行に収まる
- GitHub PagesのHTTP応答が `200 OK`
- GitHub Pages状態が `built`

## 注意事項

- このモックは静的HTMLであり、実際の認証・SMS送信・Wallet発行は行わない
- 既存DB接続、新DB保存、ログ保存、Wallet Pass発行はすべて設計説明用の注釈
- Apple Wallet公式バッジは、Apple Developerページ上で利用許諾同意が必要だったため、現時点では公式素材を同梱していない
- Google Wallet公式SVGは取得済みだが、表示文言統一のため現在はカスタム黒ボタンを使用している

## 直近の状態

最終的なログイン導線:

1. 東京都スキー連盟会員番号を入力
2. 携帯電話番号を入力
3. `携帯電話にログインコードを送る` を押す
4. `携帯電話に届いた6桁の数字を入力してください` の画面でコード確認
5. 東京都スキー連盟会員マイページへ遷移
6. デジタル会員証を確認
7. AppleウォレットまたはGoogleウォレットに登録する
